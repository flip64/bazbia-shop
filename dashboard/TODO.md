# قوانین توسعه داشبورد

1. هر صفحه فقط یک مسئولیت اصلی داشته باشد.
2. از نام‌های وابسته به کتابخانه‌ها مثل `datatable` برای مفهوم صفحه استفاده نشود.
3. جستجو، مرتب‌سازی و صفحه‌بندی تا حد ممکن سمت Django انجام شوند.
4. فایل‌های استاتیک خروجی `collectstatic` وارد Git نشوند.
5. قبل از تغییر ساختاری بزرگ، وابستگی فایل‌ها بررسی شود.
6. از حدس درباره مدل‌ها و سرویس‌ها خودداری شود.
7. تغییرات مرحله‌به‌مرحله انجام شوند.
8. بعد از پایان هر مرحله، نتیجه تست شود.
9. از Queryهای بدون `select_related` در جدول‌های بزرگ خودداری شود.
10. طراحی در موبایل و دسکتاپ هم‌زمان بررسی شود.
